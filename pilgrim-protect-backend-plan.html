API Endpoints — api.pilgrimprotectstory.org
/api/auth ▾
POST /api/auth/register Create worker or donor account
POST /api/auth/login Returns JWT token
GET /api/auth/me Current user profile
/api/schools ▾
GET /api/schools All schools (map pins)
GET /api/schools/:id School profile + spray history
POST /api/schools Add new school (admin)
PUT /api/schools/:id Update school info (admin)
/api/spray-reports ▾
POST /api/spray-reports Worker submits report + photos
GET /api/spray-reports All reports (filterable by school/worker/date)
GET /api/spray-reports/mine Worker's own submissions
PUT /api/spray-reports/:id/verify Supervisor verifies report
/api/donations ▾
POST /api/donations/checkout Create Stripe checkout session
POST /api/donations/webhook Stripe webhook handler
GET /api/donations/mine Donor's donation history
/api/stats ▾
GET /api/stats/impact Total schools, rooms sprayed, students protected
GET /api/stats/map GeoJSON of schools + latest spray status
GET /api/stats/timeline Spray activity over time (Chart.js data)
/api/upload ▾
POST /api/upload/image Upload to Cloudinary, returns URL

Rollout Plan — Build Order
Build philosophy: Data-first, bottom-up. Each phase produces something testable. Nothing gets built on fake data. Each layer feeds the next.
Phase 1 — Foundation

Express Server + MongoDB Schema + Auth

Initialize Express project with folder structure (routes, models, middleware, config)
Connect to MongoDB Atlas — set up Mongoose models for all 5 collections
Set up auth middleware — JWT tokens, password hashing (bcrypt), role-based guards
Build CRUD routes for /api/schools and /api/auth
Add Cloudinary integration for image uploads
Set up environment variables, CORS, rate limiting
Test all endpoints with Postman or Thunder Client
~1–2 weeks
✓ Deliverable: Working API you can hit from any client. Schools CRUD is live.
Phase 2 — Real Data

Populate Schools + Seed Database

Research and collect real school data — names, districts, GPS coordinates
Build a seed script (seed.js) that loads schools into MongoDB
Add geospatial index (2dsphere) on school locations for map queries
Create /api/stats/map endpoint returning GeoJSON for Mapbox
Pull initial MAP data — district-level parasite rates as context layer
Create admin route to add/edit schools without needing DB access
~1 week
✓ Deliverable: Database full of real schools with real coordinates. Map endpoint returns GeoJSON.
Phase 3 — Field Worker App

Expo Mobile App → Spray Reports

Initialize Expo project — minimal UI, 4 screens max
Login screen — authenticate against /api/auth, store JWT
School list — pull assigned schools from /api/schools
Spray report form — date, rooms sprayed, photo upload, notes
Submission history — worker's past reports from /api/spray-reports/mine
Offline queue — store submissions in AsyncStorage, sync when online
Auto GPS capture on report submission
~2 weeks
✓ Deliverable: Workers can log spray activity from phones. Data flows into MongoDB.
Phase 4 — The Map & Website

Next.js + Mapbox + School Profiles + Dashboard

Initialize Next.js 15 project with Tailwind + shadcn/ui
Build interactive Mapbox map — school markers from /api/stats/map
School popups with name, last spray date, student count, photos
School profile pages — full spray history, photo gallery, impact story
Impact dashboard — total schools, rooms, students (Chart.js)
Storytelling page — curated narratives from field data
Deploy frontend to Vercel, backend to Railway
~2–3 weeks
✓ Deliverable: Live website at pilgrimprotectstory.org showing real impact data on the map.
Phase 5 — Donations & Engagement

Stripe + Brevo + Donor Portal

Integrate Stripe Checkout — one-time and recurring donations
Apply for Stripe nonprofit rate (2.2% + $0.30)
"Sponsor a School" flow — donor picks school, pays, gets linked
Stripe webhook → create Donation record, update Donor totals
Brevo email setup — welcome email, donation receipt, monthly impact update
Donor dashboard — see sponsored schools, spray updates, donation history
Apply for Mapbox nonprofit program (free tier boost)
~2 weeks
✓ Deliverable: Full donor flow — discover → donate → track impact. Emails go out automatically.
